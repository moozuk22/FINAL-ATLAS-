# Инструкции за миграция на базата данни

## Стъпки за изпълнение на миграцията

### 1. Отворете Supabase Dashboard
- Отидете на https://supabase.com/dashboard
- Изберете вашия проект

### 2. Отворете SQL Editor
- В лявото меню кликнете на "SQL Editor"
- Кликнете на "New Query"

### 3. Копирайте и изпълнете миграцията
- Отворете файла `supabase_migrations.sql` в проекта
- Копирайте целия SQL код
- Поставете го в SQL Editor в Supabase
- Кликнете на "Run" или натиснете Ctrl+Enter

### 4. Проверете резултата
- Трябва да видите съобщение "Success. No rows returned"
- Ако има грешки, проверете дали таблиците вече съществуват

## Какво прави миграцията?

### Нови колони в съществуващи таблици:
- `table_requests.source` - Източник на заявката (nfc/qr/direct)
- `table_requests.request_type` - Тип заявка (waiter/bill/animator/order)
- `table_requests.assigned_to` - Кой е приел заявката (за аниматор)
- `menu_items.is_daily_menu` - Дали е в менюто за деня
- `menu_items.daily_menu_date` - Дата на менюто за деня
- `restaurant_tables.session_started_at` - Начало на сесията

### Нови таблици:
- `customer_ratings` - Рейтинги и feedback от клиенти
- `daily_menu_assignments` - Назначения за меню за деня

### Индекси и политики:
- Създадени индекси за по-добра производителност
- Настроени Row Level Security политики

## Важни бележки

- Миграцията използва `IF NOT EXISTS`, така че е безопасна за повторно изпълнение
- Съществуващите данни няма да бъдат загубени
- Новите колони имат default стойности

## След миграцията

1. Проверете дали всички таблици са създадени успешно
2. Тествайте функционалностите в приложението
3. Ако има проблеми, проверете логовете в Supabase
